Ch: 5 Javascript Arrays
Arrays represent a collection of items.

Arrays are a fundamental data structure. The objects stored in them can of any type, or combination of types.
They are denoted/built with [thing1,thing2,cat,hat];

The four fundamental operations of an array, or any data-structure, are:
(note: The Big(O) values below are the "should be" values, and may vary)

Insertion
Adding a new item inside a data structure.
.push(element), add element to the end of the array. O(1)
Deletion
Removing an item from the data structure.
.pop() removes the last element of the array. O(1)
Access
Access the item stored in a known point in the data structure.
array[index] returns the element at index index O(1)
Iteration
The act of going over each of the items in a data structure. Fundamental to Searching a data structure.
Most often done with a for(i; condition; increment) loop. O(n)
for(index in array). This will run a loop once for each key (indexes for an array, key's for an object) in the data structure, assigning each key to the element variable in turn.
for(value of array) will do the same, but uses the actual values stored in the data structure, rather than the keys.

Ch: 7 Javascript Memory Management
Memory Management Notes

A memory leak is a failure to release discarded memory. Left unchecked, this process may consume all available membory.

  • A reference to an Object, or property therof, loads the entire object; not just the parts you use.
  • A var pointing to DOM element will leak if the DOM element is deleted. ie:
    ---.html---
    <div id="thatDiv">
    ------------
    ---.js------
    var thatDiv = document.quereySelector('#thatDiv'); thatDiv.remove(); //Removes it from the DOM, but it is still taking up memory. ------------

    This can be avoided by ensuring that the reference is confined to the scope in which it is used.
  • Anything in the global window object, is in memory, and will not be garbage collected.
  • An object is cleared when all refrences to that object are cleared. Thus it is good practice to limit object references when possible.
  • The delete operator will remove unwanted object propertys. delete will have no effect on non-objects.
